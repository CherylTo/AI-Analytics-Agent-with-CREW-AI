# 🤖 AgentAnalytics

An AI-powered business intelligence system built with [CrewAI](https://crewai.com) and [DuckDB](https://duckdb.org), using the [Olist Brazilian E-Commerce dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce). Ask natural language questions about the data and get SQL queries, charts, and business insights — all generated by a crew of AI agents.

---

## Overview

AgentAnalytics uses a multi-agent architecture to answer business questions end-to-end:

1. You ask a question in plain English
2. A **Manager** agent interprets it and creates a structured query plan
3. A **SQL Generator** writes a DuckDB SQL query
4. The query is executed against a local DuckDB database
5. A **Visualizer** selects the best chart type and prepares metadata
6. An **Insight Analyst** generates a plain English summary and key takeaway
7. Results are displayed in a Streamlit dashboard

---

## Architecture

```
User Question
      │
      ▼
┌─────────────┐
│   Manager   │  Interprets question → QueryPlan
└──────┬──────┘
       │
       ▼
┌──────────────────┐
│  SQL Generator   │  QueryPlan → DuckDB SQL
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│   execute_query  │  SQL → Pandas DataFrame  (outside crew)
└──────┬───────────┘
       │
       ▼
┌─────────────┐
│  Visualizer │  DataFrame → ChartMetadata
└──────┬──────┘
       │
       ▼
┌──────────────────┐
│ Insight Analyst  │  DataFrame + ChartMetadata → InsightOutput
└──────────────────┘
       │
       ▼
  Streamlit Dashboard
  (chart + table + insight)
```

**Two-phase crew execution:**

- `crew_sql` — Manager + SQL Generator (hierarchical process with validation)
- `crew_insights` — Visualizer + Insight Analyst (hierarchical process with validation)

## Setup

### Prerequisites

- Python 3.12+
- [uv](https://docs.astral.sh/uv/) package manager
- OpenAI API key

### 1. Clone the repo

```bash
git clone https://github.com/your-username/AgentAnalytics.git
cd AgentAnalytics
```

### 2. Create your `.env` file

```bash
touch .env
```

Add your OpenAI API key:

```
OPENAI_API_KEY=sk-...
```

### 3. Install dependencies

```bash
uv sync
```

### 4. Download the dataset

Download the [Olist Brazilian E-Commerce dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) from Kaggle and place all CSV files into the `data/` folder.

### 5. Set up the database

```bash
uv run python scripts/db_setup.py
```

This loads the CSV files into DuckDB and creates the `analytics_orders` view.

---

## Running from the Terminal

Run from the project root:

```bash
PYTHONPATH=ai_analytics_agent/src uv run python -m ai_analytics_agent.main "Which product categories generate the most revenue?"
```

This runs the full agent crew and saves a markdown report to `ai_analytics_agent/output/ai_analytics_report.md`.

---

## Running the Streamlit App

```bash
streamlit run streamlit/app.py
```

Open your browser at `http://localhost:8501`.

The app has two tabs:

- **Overview** — project description, data model, and how it works
- **Ask a Question** — type a question or pick from the preloaded list, click Analyze, and see the chart, data table, and AI insight

## Dataset

The [Olist Brazilian E-Commerce dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) contains ~100k orders from 2016–2018 across multiple marketplaces in Brazil. It includes order status, pricing, payments, delivery performance, customer locations, and product reviews.

All data is joined into a single `analytics_orders` view with 16 columns covering orders, customers, sellers, products, payments, and delivery metrics.

---

## Notes

- The crew uses `Process.hierarchical` with a manager agent that validates SQL and chart metadata before proceeding
- SQL execution happens outside the crew between the two phases, keeping the agents focused on reasoning rather than execution
- All structured outputs use Pydantic models for type safety
- The Streamlit app runs independently from the CLI — both call `run_ai_analytics()` directly
- If the SQL generator returns prose instead of SQL, the system automatically retries up to 3 times
